📊 TECHNICAL INFO
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Database:
─────────
Table: Employee Directory
Field: Announcement (TEXT, stores JSON)
Format: {"type":"...", "title":"...", "message":"..."}

Query (Admin Creates):
──────────────────────
UPDATE "Employee Directory"
SET Announcement = '{"type":"motivation","title":"...","message":"..."}'
WHERE status = 'Active';

Query (Admin Deletes):
──────────────────────
UPDATE "Employee Directory"
SET Announcement = NULL
WHERE status = 'Active';

Query (Load Announcement):
──────────────────────────
SELECT Announcement 
FROM "Employee Directory"
WHERE status = 'Active' AND Announcement IS NOT NULL
LIMIT 1;

❓ FAQ
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Q: Do I need to run any SQL migrations?
A: NO! It uses your existing Employee Directory table.

Q: Will all employees see the same announcement?
A: YES! When admin creates/edits, it updates ALL active employees.

Q: Can employees edit announcements?
A: NO, only admins can create/edit/delete.

Q: How long does dismissal last?
A: 24 hours, then the banner reappears automatically.

Q: Can I have different announcements for different employees?
A: Currently no, all active employees see the same announcement.
   (This keeps it simple and consistent!)

Q: What if an employee is not Active?
A: Inactive employees won't get announcements. Only status='Active' 
   employees are updated.
